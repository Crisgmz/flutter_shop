====================================================
1) RESUMEN EJECUTIVO
====================================================
El sistema actual en React es un ERP/POS para RD con enfoque en:
- Punto de venta
- Inventario
- Compras
- Clientes/Proveedores
- Cobros/Caja
- Reportes/Impuestos
- Configuracion operativa

Estado actual:
- UI funcional en todas las secciones principales.
- Datos en memoria (mock/local state), sin backend productivo conectado.
- Se agregaron handlers a botones clave que estaban solo visuales.
- Se agregaron utilidades reutilizables para no duplicar logica.

Objetivo de migracion:
- Rehacer frontend en Flutter (Web + Android inicialmente) con backend Supabase.
- Mantener UX/flujo actual y evolucionar a persistencia real, roles, auditoria y fiscal.

====================================================
5) PRD (PRODUCT REQUIREMENTS DOCUMENT) - VERSION MIGRACION FLUTTER
====================================================
5.1 Problema
El sistema actual opera con UI funcional pero datos locales; se necesita una plataforma productiva en Flutter con backend Supabase para operacion real multiusuario.

5.2 Objetivo
Migrar frontend a Flutter conservando los flujos existentes y habilitando persistencia, seguridad, roles y trazabilidad.

5.3 Usuarios
- Administrador
- Cajero
- Supervisor
- Contador

5.4 Objetivos de negocio
- Reducir tiempo de venta en caja.
- Mejorar control de inventario y cuentas por cobrar/pagar.
- Centralizar reportes e informacion fiscal.

5.5 Requerimientos funcionales (MVP)
1. Autenticacion
- Login con email/password (Supabase Auth)
- Cierre de sesion

2. Inventario
- CRUD de productos
- Busqueda/filtros
- Stock y alertas de bajo inventario

3. Ventas POS
- Crear venta con items
- Calcular subtotal/impuestos/total
- Registrar cobro (contado/credito)

4. Clientes
- CRUD clientes
- Balance por cobrar

5. Cobros
- Registrar abonos a facturas
- Historial de pagos

6. Compras y Proveedores
- CRUD proveedores
- Registro de compras

7. Caja
- Apertura/cierre de caja
- Diferencias

8. Reportes base
- Ventas por periodo
- Exportes CSV

5.6 Requerimientos no funcionales
- Tiempo de respuesta UI < 300ms para interacciones comunes
- Soporte Web y Android en MVP
- Manejo de errores de red y reconexion
- Seguridad por roles en frontend + RLS en Supabase

5.7 Fuera de alcance inicial (post-MVP)
- Integracion fiscal DGII completa (generacion final certificada)
- Impresion fiscal avanzada
- Multiempresa complejo

5.8 KPIs de exito
- 95% de acciones de POS sin errores
- <2% de ventas con inconsistencias de total/inventario
- 100% de operaciones auditables (created_by/updated_by)
